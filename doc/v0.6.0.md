# v0.6.0 版本说明

这一版主要把 RAG 的知识库接入方式改成了更接近真实业务的增量索引模式。

---

## 这版优化了什么

1. 改成使用本机 PostgreSQL + pgvector
2. 去掉每次启动都重复切片入库的做法
3. 新增知识库索引状态表
4. 支持按文件 hash 判断是否需要重新向量化
5. 删除的文档会同步清理旧向量

---

## 实现原理

### 1. 首次全量

第一次启动时，系统会读取 `knowledge/` 目录下的文档，完成：

- 读取文档
- 切片
- 生成 embedding
- 写入 pgvector
- 写入索引状态表

### 2. 后续增量

后续启动时，系统会重新扫描 `knowledge/`：

- 如果文件内容 hash 没变，直接跳过
- 如果文件内容变了，先删旧向量，再重新入库
- 如果文件删掉了，也会清掉对应向量和索引记录

### 3. 为什么这样做

这样可以避免：

- 每次启动都重复切片
- 向量库里出现重复数据
- 知识库越跑越乱

---

## 运行流程

```text
启动应用
-> 扫描 knowledge/
-> 计算文件 hash
-> 对比索引表
-> 决定新增 / 更新 / 删除
-> 同步到 pgvector
-> 在线请求时做检索
-> 交给 LLM 回答
```

---

## 验证方法

1. 新增一个 md 文件，重启项目，看是否会新增对应向量
2. 修改一个已有 md 文件，重启项目，看旧向量是否被替换
3. 删除一个 md 文件，重启项目，看旧向量是否被清理
4. 不修改任何文件，重启项目，看是否会直接跳过索引

---

## 这版适合学习什么

- 启动期任务和在线检索的分层
- 向量库的增量维护
- 文件 hash 去重
- 用 metadata 管理 RAG 文档来源
