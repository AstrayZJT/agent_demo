# 版本更新记录

> 这是项目当前学习版本的更新说明。后续新增功能时，可以继续按版本往下追加。

## 版本号规则

从这一版开始，后续版本统一使用语义化版本号：

- `v主版本.次版本.修订号`
- 主版本：结构性重构或不兼容变更
- 次版本：新增功能、模块、流程
- 修订号：bug 修复、文档补充、配置优化

当前建议使用的版本标记是：

- `v0.5.0`

说明：

- 目前项目还处在学习和演进阶段，所以先用 `0.x.y`
- 以后新增功能就按 `v0.5.1`、`v0.6.0` 这样递增
- 之前的 `v1-baseline`、`v1-memory` 这类 tag 保留为历史里程碑，不强行重命名

## 当前版本概览

这一版主要把项目从“能跑的 AI Demo”优化成了一个更完整的学习型 Agent：

- 可以按请求决定是否使用 RAG
- 可以按 `userId` 做记忆隔离
- 可以使用工具调用做联网查询
- 加入了护栏机制，提升输入输出安全性
- 补齐了中文文档和学习说明

## 本版的运行特点

### RAG 路线

请求进入后：

1. 读取 `knowledge/`
2. 切片
3. 向量化
4. 检索相关内容
5. 把检索到的片段交给 LLM

### 工具调用路线

请求进入后：

1. 模型决定是否调用 `webSearch` / `visitWebpage`
2. 工具执行并返回结果
3. 模型基于工具结果组织回答

### 记忆路线

请求进入后：

1. 根据 `userId` 读取历史消息
2. 拼接上下文
3. 交给模型
4. 回答后再写回数据库

## 如何验证这版优化

- 测 RAG：传 `useRag=true`
- 测工具调用：传 `useRag=false`
- 测匿名请求：传 `userId=-1`
- 测护栏：输入明显注入式内容，看是否被拦截
- 测记忆：同一个 `userId` 连续问两次，看上下文是否延续
