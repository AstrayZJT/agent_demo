# v0.6.1 版本说明

这一版主要做了上下文压缩，让项目更像一个真正会“收纳记忆”的学习型 Agent。

---

## 这版改了什么

1. `MessageWindowChatMemory` 改成 `TokenWindowChatMemory`
2. `buildMemoryContext()` 改成摘要输出
3. system prompt 只保留规则和摘要
4. 最近消息保留原文
5. RAG 继续保持小 topK 和较稳阈值

---

## 实现原理

### 1. token 窗口记忆

原来是按消息条数保留，现在改成按 token 预算保留。

这样做的好处是：

- 更符合模型实际上下文上限
- 中文长文本更容易控制
- 不会因为一句长消息把窗口挤爆

### 2. 历史摘要

更早的消息不再完整返回，而是压缩成摘要放进 system prompt。

### 3. 最近原文

最近几轮消息仍然保留原文，避免丢掉当前对话状态。

### 4. prompt 更干净

system prompt 只保留：

- 规则
- 历史摘要
- 最近原文

这样模型看到的上下文更稳定，也更省 token。

---

## 运行流程

```text
用户请求
-> 读取历史消息
-> 更早内容压成摘要
-> 最近消息保留原文
-> TokenWindowChatMemory 保留当前窗口
-> system prompt 拼规则 + 摘要
-> LLM 生成回答
-> 回写数据库
```

---

## 你可以怎么验证

1. 连续对话 6 轮以上，看早期内容是否逐渐变成摘要
2. 看 system prompt 是否比以前短
3. 看最近几轮是否还能被模型正常引用
4. 观察长对话时是否更稳定

---

## 这版适合学习什么

- token 预算管理
- prompt 压缩
- 聊天记忆分层
- 长对话上下文控制
